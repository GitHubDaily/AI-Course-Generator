## 一、产品概述

### 1.1 产品定位

一款基于 AI 技术的教育工具，帮助小学教师快速从课本 PDF 生成结构化的课程大纲和教学内容，提升备课效率。

### 1.2 目标用户

**主要用户：**
- 小学教师（1-6年级）
- 教研人员
- 教育机构内容团队

**用户画像：**
- 年龄：25-45岁
- 工作场景：需要频繁备课
- 痛点：备课时间长、课程设计不系统
- 技术水平：基础电脑操作能力

### 1.3 核心价值主张

**对用户：**
- ⏱️ 将备课时间从 2-3 小时缩短到 10 分钟
- 📊 提供标准化、结构化的课程框架
- 🎯 AI 智能提取教学重点难点
- 📱 简单易用，无需学习成本

**对业务：**
- 🚀 快速验证 AI+教育的市场需求
- 💰 为后续付费功能打基础
- 📈 积累教育内容数据

### 1.4 产品目标

**MVP 阶段（2周内）：**
- 完成核心功能：PDF 上传 → 课程大纲生成 → 结果展示
- 支持至少 3 个学科（语文、数学、英语）
- 课程生成成功率 ≥ 85%
- 用户可用性测试满意度 ≥ 80%

**V1.0 阶段（1个月内）：**
- 新增详细教学内容生成
- 支持多种导出格式
- 用户留存率 ≥ 60%

## 三、详细功能需求

### 3.1 文件上传模块

#### 3.1.1 PDF 上传功能

**功能描述：**
用户通过拖拽或点击方式上传小学课本 PDF 文件。

**交互流程：**
```
1. 用户进入首页
2. 看到上传区域（虚线框 + 提示文字）
3. 拖拽 PDF 文件到上传区 OR 点击选择文件
4. 系统显示文件名和大小
5. 自动开始上传和处理
6. 显示进度条和处理状态
```

**功能规则：**

| 规则项 | 限制 |
|--------|------|
| 文件格式 | 仅支持 .pdf |
| 文件大小 | 最大 20MB |
| 页数限制 | 最多 50 页 |
| 单次上传 | 仅支持单个文件 |
| 并发限制 | 同一用户同时只能处理 1 个文件 |

**异常处理：**
- 文件格式错误 → 提示"仅支持 PDF 格式"
- 文件过大 → 提示"文件不能超过 20MB"
- 上传失败 → 提示"上传失败，请重试"
- 网络超时 → 提示"网络超时，请检查网络后重试"

**UI 要求：**
- 上传区域明显，占据页面中心位置
- 拖拽时高亮显示上传区域
- 显示支持的文件格式和大小限制
- 上传后显示文件信息（名称、大小）

#### 3.1.2 文本提取功能

**功能描述：**
从上传的 PDF 中自动提取文本内容。

**技术方案：**
1. **方案A：直接提取**（电子版 PDF）
   - 使用 PDF 解析库直接提取文本
   - 适用于可复制文字的 PDF
   - 速度快，准确度高

2. **方案B：OCR 识别**（扫描版 PDF）
   - 将 PDF 转图片后 OCR 识别
   - 适用于扫描件或图片 PDF
   - 速度较慢，需要后备方案

**处理流程：**
```
1. 上传完成后自动开始提取
2. 先尝试方案A（直接提取）
3. 如果提取内容少于50字，自动切换方案B
4. 提取完成后预览前 500 字
5. 用户可查看完整提取结果
```

**质量保证：**
- 提取后去除页眉页脚
- 去除多余空格和换行
- 保留段落结构
- 识别章节标题

**UI 要求：**
- 显示"正在提取文本..."加载状态
- 提供文本预览（可展开查看完整内容）
- 如果提取质量差，提示用户确认

### 3.2 AI 处理模块

#### 3.2.1 工作流 1：内容分析

**功能目标：**
分析课本内容，提取关键信息和教学要点。

**输入：**
- 提取的课本文本内容（字符串）
- 年级信息（可选）
- 学科信息（可选）

**处理内容：**
1. 识别课本的章节结构
2.提取每章的核心知识点（3-5个）
3. 标注教学重点和难点
4. 生成内容概要

**输出：**
结构化的内容分析结果（JSON 格式）：
```json
{
  "subject": "学科名称",
  "grade": "年级",
  "chapters": [
    {
      "chapter_number": 1,
      "title": "章节标题",
      "main_content": "主要内容摘要",
      "key_points": ["知识点1", "知识点2", "知识点3"],
      "difficulty_points": ["难点1", "难点2"]
    }
  ],
  "overall_summary": "整体内容概述"
}
```

**质量要求：**
- 知识点提取准确率 ≥ 90%
- 章节识别准确率 ≥ 95%
- 处理时间 < 30秒

#### 3.2.2 工作流 2：课程大纲生成（核心功能）⭐

**功能目标：**
根据内容分析结果，生成结构化的课程大纲。

**输入：**
- 工作流1的内容分析结果
- 期望的模块数量（默认 3-4 个）
- 每课时长（默认 40 分钟）

**处理内容：**
1. 将内容划分为 3-4 个学习模块
2. 为每个模块生成：
   - 模块标题（简洁、有吸引力）
   - 模块描述（30-50字）
   - 3-5个学习目标（符合 SMART 原则）
   - 3个核心关键概念
3. 确保模块之间有逻辑递进关系
4. 适配小学生认知水平

**输出：**
完整的课程大纲（JSON 格式）：
```json
{
  "course_title": "课程标题",
  "grade": "年级",
  "subject": "学科",
  "total_modules": 4,
  "estimated_hours": 4,
  "modules": [
    {
      "module_id": "M001",
      "title": "模块标题",
      "description": "模块描述文字",
      "sequence": 1,
      "duration_minutes": 80,
      "learning_objectives": [
        "学习目标1",
        "学习目标2",
        "学习目标3"
      ],
      "key_concepts": [
        "关键概念1",
        "关键概念2",
        "关键概念3"
      ],
      "prerequisites": []
    }
  ]
}
```

**质量要求：**
- 模块划分合理性 ≥ 85%
- 学习目标明确性 ≥ 90%
- 用户满意度 ≥ 80%
- 生成时间 < 60秒

**业务规则：**
- 每个模块时长控制在 40-90 分钟
- 学习目标数量：3-5个
- 关键概念数量：固定3个
- 模块数量：3-5个（推荐4个）

#### 3.2.3 工作流 3：详细内容生成（V1.0）

**功能目标：**
为每个模块生成详细的教学内容。

**输入：**
- 单个模块信息
- 相关的课本原文内容

**处理内容：**
1. 生成教学计划（4个环节）：
   - 导入环节（5分钟）
   - 主要内容（25分钟）
   - 练习环节（8分钟）
   - 总结环节（2分钟）
2. 生成 2-3 个教学案例
3. 生成 5-10 道练习题（含答案和解析）
4. 提供 3-5 条教学建议

**输出：**
详细教学内容（JSON 格式）

**质量要求：**
- 教学流程合理性 ≥ 85%
- 练习题质量 ≥ 80%
- 案例相关性 ≥ 85%

## 五、技术方案

### 5.1 技术架构

**前端技术栈：**
- 框架：React 18.x
- UI 库：Ant Design 5.x
- 语言：TypeScript
- 状态管理：React Query
- 路由：React Router

**后端技术栈：**
- 框架：FastAPI（Python）
- PDF 处理：PyPDF2 / pdfplumber
- HTTP 客户端：httpx（异步）
- 数据验证：Pydantic

**AI 平台：**
- 讯飞星辰 Agent（工作流引擎）
- 讯飞星火大模型（V3.5）

**部署方案：**
- 前端：Vercel / Netlify
- 后端：阿里云 / 腾讯云
- 数据库：PostgreSQL（V1.0 需要）

### 5.2 工作流实现方案

**平台选择：讯飞星辰 Agent 云平台**

**工作流配置：**

1. **工作流1：TextbookContentAnalysis**
   - 输入参数：TEXTBOOK_CONTENT（文本）
   - 核心节点：大模型分析节点
   - 输出：结构化的内容分析结果

2. **工作流2：CourseOutlineGeneration**
   - 输入参数：CONTENT_ANALYSIS（JSON）
   - 核心节点：大模型生成节点
   - 输出：课程大纲（JSON）

3. **工作流3：DetailedContentGeneration**
   - 输入参数：MODULE_INFO（JSON）
   - 核心节点：大模型生成节点
   - 输出：详细教学内容（JSON）

**Prompt 工程：**
- 使用详细的 system prompt
- 提供明确的输出格式示例
- 加入质量控制指令

### 5.3 PDF 处理方案

**方案选择：**
- 优先使用纯文本提取（PyPDF2）
- 降级使用 OCR（讯飞 OCR API）

**处理流程：**
1. 验证文件格式和大小
2. 尝试文本提取
3. 如果提取失败或内容太少，使用 OCR
4. 文本清洗和格式化
5. 返回结构化结果

## 六、数据结构设计

### 6.1 核心数据模型

```
Course Outline
├── course_title (string)          # 课程标题

        ├── key_concepts (array)         # 关键概念
        └── prerequisites (array)        # 前置要求
```

```
Detailed Content
├── module_id (string)             # 关联的模块ID

│   ├── introduction (object)      # 导入环节
│   ├── main_content (object)      # 主要内容

│   └── summary (object)           # 总结环节
├── examples (array)               # 教学案例
├── exercises (array)              # 练习题
│   └── Exercise
│       ├── id (string)
│       ├── type (string)          # 题型
│       ├── question (string)      # 题目
│       ├── answer (string)        # 答案
│       ├── difficulty (string)    # 难度
```

## 七、用户流程

### 7.1 核心用户流程

```
[用户进入首页]
      ↓
[查看产品介绍]
      ↓
[上传 PDF 文件] → [文件验证] → 失败 → [提示错误，重新上传]
      ↓ 成功
[显示上传进度]
      ↓
[文本提取中...]
      ↓
[预览提取的文本] → 用户确认质量
      ↓
[AI 生成课程大纲...]
      ↓
[显示生成进度]
  - ✓ 内容分析完成
  - ⏳ 大纲生成中...
      ↓
[展示课程大纲]
  - 查看所有模块
  - 查看单个模块详情
  - 导出大纲
      ↓
[用户满意] → [导出或分享] → [结束]
      ↓ 或
[用户不满意] → [重新上传] → [重新开始]
```

### 7.2 异常流程处理

**场景1：上传失败**
```
上传 → 失败 → 显示错误原因 → 提供重试按钮
```

**场景2：提取文本质量差**
```
提取完成 → 检测质量 → 质量差 → 提示用户确认 
→ 用户选择：继续 / 重新上传
```

**场景3：生成超时**
```
生成中 → 超过2分钟 → 显示"生成时间较长，请继续等待" 
→ 超过5分钟 → 提示失败，建议重试
```

**场景4：生成结果不满意**
```
查看结果 → 不满意 → 重新生成（使用同一文件）
→ 调整参数（如模块数量）→ 重新生成
```

## 十四、FAQ

### Q1: 支持哪些类型的 PDF？
**A:** 支持电子版 PDF 和扫描版 PDF。电子版提取速度更快、准确度更高；扫描版会使用 OCR 技术识别，可能需要更长时间。

### Q2: 生成的课程大纲准确吗？
**A:** AI 生成的大纲基于课本内容分析，准确率约 85-90%。建议教师根据实际情况进行调整。后续版本会持续优化。

### Q3: 一次可以上传多少个文件？
**A:** MVP 版本仅支持单个文件上传。V1.0 会考虑批量上传功能。

### Q4: 生成失败怎么办？
**A:** 系统会提示具体原因（如文件格式错误、内容无法识别等）。可以尝试重新上传或更换文件。

### Q5: 生成的内容可以商用吗？
**A:** 生成的内容供教师个人教学使用。商业用途请查看用户协议或联系我们。

### Q6: 数据安全吗？
**A:** 所有上传的文件加密存储，7天后自动删除。不会用于其他用途。
